【为什么 Line 更好？（写报告时的理由）】
================================================================================

原因 1：能感知方向（关键！）
  - Line：能检测 Leader 向左/右偏移 → 知道往哪边转
  - Bump：只知道距离，不知道方向 → 无法预测曲线
  → Line 能提前感知曲线方向

原因 2：预测性控制
  - Line：提前检测到曲线，主动调整方向
  - Bump：发现偏离后才调整，总是滞后
  → Line 响应及时，Bump 来不及

原因 3：比例反馈控制
  - Line：偏离越多，转向越大（合理调整）
  - Bump：转向信号弱，容易判断错误
  → Line 转向幅度合理，Bump 容易转错

【常见问题解答（给组员）】
================================================================================

Q1: 为什么 Bump Sensor 在曲线上这么差？
A: 因为它只能测量距离，不能感知 Leader 往左还是往右转。就像蒙着眼
   开车转弯，只知道前面有东西，不知道弯往哪边，结果就乱转。

Q2: 为什么 Line Sensor 在曲线上这么好？
A: 因为它能检测 Leader 的横向位置变化，提前知道要往哪边转，可以
   主动调整方向，不会等到偏离很大才反应。

Q3: 为什么假设 A 和假设 B 的结论完全相反？
A: 因为不同任务需要不同的传感器能力：
   - 直线：距离控制最重要 → Bump 的线性测距优势
   - 曲线：方向控制最重要 → Line 的横向检测优势

Q4: 这个结果能用在报告的哪里？
A: 可以用在：
   - 实验结果章节（展示曲线数据）
   - 对比分析章节（与假设 A 对比）
   - 讨论章节（分析原因，引出假设 C）
   - 结论章节（总结发现）

Q5: 为什么需要假设 C（混合传感器）？
A: 因为：
   - 假设 A 证明：Bump 适合直线，Line 不适合
   - 假设 B 证明：Line 适合曲线，Bump 不适合
   - 实际路径：既有直线又有曲线
   - 解决方案：结合两者优势 → Bump+Line Hybrid（假设 C）
