================================================================================
图表解释：Line Sensor vs Bump Sensor 30°曲线轨迹对比图
================================================================================

【这张图在做什么？】
这张图展示了两个机器人跟随系统在走 30° 曲线时的表现对比：
- 左边：使用 Line Sensor（绿色线条）- 表现很好
- 右边：使用 Bump Sensor（红色线条）- 表现很差

【实验设置】
- 任务：一个跟随机器人（Follower）跟着领导机器人（Leader）走 30° 曲线
- 理想情况：跟随机器人应该走出平滑的曲线轨迹
- 实际情况：每个传感器做了 10 次重复实验，每条线代表一次实验

================================================================================
【上半部分：轨迹图 - 看机器人能不能走好曲线】
================================================================================

左上图（绿色 - Line Sensor）：✓ 很优秀
--------------------------------------------------
优点 1：曲线平滑流畅
  - 10 条绿色线条呈现出漂亮的曲线形状
  - 从 Y=0 逐渐平滑下降到 Y=-70mm 左右
  - 就像一条优美的抛物线，非常自然

优点 2：轨迹一致性好
  - 10 条线紧密聚集在一起
  - 每次实验的曲线形状都很相似
  - 说明系统稳定可靠，可重复性强

优点 3：能够预测曲线
  - 轨迹逐渐弯曲，没有突然的转向
  - 说明 Line Sensor 能提前感知曲线，平滑调整方向
  - 就像一个有经验的司机，提前打方向盘

右上图（红色 - Bump Sensor）：❌ 很糟糕
--------------------------------------------------
问题 1：轨迹混乱无序
  - 10 条红色线条到处乱飘
  - 有的往上飘（+180mm），有的往下（-170mm）
  - Y 轴范围达到 350mm！是 Line Sensor 的 5 倍

问题 2：无法形成稳定曲线
  - 看不出明确的曲线形状
  - 有的实验轨迹向上弯，有的向下弯
  - 完全无法预测下次会走成什么样

问题 3：反应式跟随失败
  - Bump Sensor 只能感知距离，无法预测曲线方向
  - 等发现偏离时已经来不及调整
  - 就像开车只看前方 1 米，根本看不到弯道

为什么差这么多？
  - Line Sensor：能检测到 Leader 的横向位置偏移 → 提前知道要转弯
  - Bump Sensor：只能检测距离，不知道 Leader 往哪边转 → 反应慢

================================================================================
【下半部分：航向角图 - 看机器人转向稳不稳】
================================================================================

左下图（绿色 - Line Sensor）：✓ 平滑转向
--------------------------------------------------
优点：航向角逐渐变化
  - 航向角从 0° 平滑过渡到 -40°
  - 变化曲线很平滑，没有突然的跳变
  - 10 条线的走势基本一致

为什么这很好？
  - 平滑的航向变化 = 平滑的转向控制
  - 机器人像在优雅地转弯，不是急打方向盘
  - 这种平滑控制才能走出流畅的曲线

虚线（Heading=0）的意义：
  - 代表直线前进的方向
  - Line Sensor 逐渐偏离这条线，说明在稳定转弯

右下图（红色 - Bump Sensor）：❌ 转向混乱
--------------------------------------------------
问题：航向角剧烈波动
  - 航向角在 -60° 到 +60° 之间疯狂摆动
  - 有的实验突然跳到 +60°（完全转错方向！）
  - 波动范围是 Line Sensor 的 3 倍

为什么这很糟糕？
  - 航向不稳 → 转向不确定 → 无法走出稳定曲线
  - 有时向左转，有时向右转，完全失控
  - 就像方向盘被随机打，车到处乱转

特别严重的问题：
  - 有一条线到达 +60°（正值）
  - 说明这次实验中机器人转错了方向！
  - 应该向下转（负值），却向上转了（正值）

【一句话总结】
================================================================================

Line Sensor（绿色）：
  ✓ 曲线平滑优美，转向稳定可控，非常适合走曲线

Bump Sensor（红色）：
  ❌ 轨迹混乱无序，转向失控乱转，完全不适合走曲线

数据对比：
  - 横向偏差范围：Line ±35mm，Bump ±175mm（相差 5 倍！）
  - 航向稳定性：Line -40° 到 0°，Bump -60° 到 +60°（相差 3 倍！）

结论：在曲线跟随任务中，Line Sensor 完胜 Bump Sensor！

================================================================================
【为什么会这样？】
================================================================================

Line Sensor 的优势（曲线跟随）：
  1. 能感知横向位置：检测到 Leader 向左/右偏移
     → 提前知道要转弯，主动调整方向
  
  2. 提供方向信息：左右传感器信号差异大时知道转向
     → 可以预测性控制，平滑转向
  
  3. 比例反馈控制：偏离越多，转向越大
     → 转向幅度合理，不会过度或不足

Bump Sensor 的劣势（曲线跟随）：
  1. 只能感知距离：只知道远近，不知道左右
     → 无法预测曲线方向
  
  2. 反应式控制：发现偏离后才开始调整
     → 调整总是滞后，来不及
  
  3. 方向判断弱：左右传感器差异小
     → 转向信号微弱，容易判断错误

简单比喻：
  - Line Sensor：像导航系统，提前告诉你前方有弯道
  - Bump Sensor：像盲人摸象，只能摸到前面有东西，不知道在哪边

【如何看懂这张图？】
================================================================================

第一步：看颜色
  - 绿色 = Line Sensor = 好（曲线上）
  - 红色 = Bump Sensor = 差（曲线上）

第二步：看轨迹形状（上排）
  - 平滑曲线 = 好（Line）
  - 混乱乱飘 = 差（Bump）

第三步：看轨迹一致性
  - 10 条线聚集 = 稳定（Line）
  - 10 条线分散 = 不稳定（Bump）

第四步：看航向角变化（下排）
  - 平滑过渡 = 转向好（Line）
  - 剧烈波动 = 转向差（Bump）

第五步：比较左右两图
  - 哪边的曲线更平滑？→ Line（左边）
  - 哪边的轨迹更一致？→ Line（左边）
  - 哪边的航向更稳定？→ Line（左边）
  - 结论：Line Sensor 完胜！

【特别注意：这与直线跟随完全相反！】
================================================================================

直线跟随（假设 A）：
  - Bump Sensor 好 ✓
  - Line Sensor 差 ❌

曲线跟随（假设 B）：
  - Line Sensor 好 ✓
  - Bump Sensor 差 ❌

为什么会反过来？
  - 直线：只需要保持距离 → Bump 的线性测距优势发挥
  - 曲线：需要感知方向 → Line 的横向位置检测优势发挥

这说明：
  - 单一传感器都有局限性
  - 需要根据任务选择合适的传感器
  - 或者...结合两者的优势（这就是假设 C 要做的！）
