图表解释：Line Sensor vs Bump Sensor 30°曲线四图详细对比
================================================================================

【这四张图在做什么？】
这是对 Line Sensor（绿色）和 Bump Sensor（红色）在 30° 曲线跟随中的全方位对比：
- 上排：Y 轴位置分布（箱线图）- 看偏离曲线的程度
- 下排：轨迹回归分析（散点图）- 看整体趋势和一致性

每个传感器都做了 10 次重复实验，这四张图从不同角度展示数据。

================================================================================
【左上图：Line Sensor Y 轴位置分布】✓ 很优秀
================================================================================

这是什么图？
  - 箱线图（Box Plot）：统计学中常用的数据分布图
  - X 轴：实验编号（1-10 次实验）
  - Y 轴：Y 位置（mm）- 机器人在曲线上的横向位置

如何看懂箱线图？
  - 箱子（绿色方块）：中间 50% 的数据集中在这里
    → 箱子越小 = 数据越集中 = 性能越稳定
  - 箱子中间的横线：中位数
  - 上下两条线（须）：数据的最大/最小值范围
  - 圆点：异常值（特别偏离的数据点）

Line Sensor 的优点：
--------------------------------------------------
优点 1：箱子很小
  - 每个实验的箱子都很小（高度 10-15mm）
  - 说明：同一次实验中，机器人轨迹很稳定
  - 就像：沿着一条固定的曲线走，不左右摇摆

优点 2：箱子位置有规律
  - 箱子从上往下逐渐移动（从 -5mm 到 -40mm）
  - 这个变化符合 30° 曲线的形状
  - 说明：机器人在按照预期的曲线轨迹移动

优点 3：偏差范围合理
  - 最小值：-50mm 左右
  - 最大值：0mm 附近
  - 整体范围约 50mm，可控

优点 4：很少异常值
  - 只有少数圆点（异常值）
  - 说明：系统稳定，很少出现突发偏差

箱子向下移动的意义：
  - 这是 30° 曲线的特征
  - 随着前进（实验 1→10），Y 位置应该逐渐减小（向下弯）
  - Line Sensor 的箱子完美体现了这个趋势！

================================================================================
【右上图：Bump Sensor Y 轴位置分布】❌ 很糟糕
================================================================================

Bump Sensor 的问题：
--------------------------------------------------
问题 1：箱子巨大
  - 每个实验的箱子都很大（高度 40-80mm）
  - 是 Line Sensor 箱子的 5 倍！
  - 说明：同一次实验中，机器人到处乱飘

问题 2：箱子位置混乱
  - 实验 1：箱子在 -100mm 附近
  - 实验 2：箱子在 -100mm 附近
  - 实验 8：箱子突然跳到 +120mm！（完全反了）
  - 没有规律，完全无法预测

问题 3：偏差范围巨大
  - 最小值：-180mm
  - 最大值：+180mm
  - 整体范围达到 360mm！是 Line Sensor 的 7 倍

问题 4：大量异常值
  - 很多圆点（异常值）出现
  - 说明：经常出现突然的大偏差
  - 系统极不稳定

最严重的问题：
  - 实验 8 的箱子在正值区域（+50mm 到 +150mm）
  - 这意味着机器人转错了方向！
  - 应该向下转（负值），却向上转了（正值）

对比总结：
--------------------------------------------------
箱子大小：Bump 的箱子是 Line 的 5 倍
箱子规律：Line 有规律下降，Bump 混乱无序
偏差范围：Bump 大 7 倍
异常值：Bump 多得多

结论：Bump Sensor 在曲线上完全失控！

【左下图：Line Sensor 轨迹回归分析】✓ 数据清晰
================================================================================

这是什么图？
  - 散点图（Scatter Plot）+ 回归线（绿色粗线）
  - X 轴：X 位置（mm）- 机器人前进的距离
  - Y 轴：Y 位置（mm）- 机器人的横向位置
  - 每个点：一个数据采样点
  - 绿色粗线：数据的整体趋势（回归线）
  - 绿色阴影：95% 置信区间（数据分布范围）

如何看懂这张图？
  - 点越集中 = 性能越好
  - 回归线越平滑 = 轨迹越稳定
  - 阴影区越窄 = 数据越一致

Line Sensor 的优点：
--------------------------------------------------
优点 1：点很集中
  - 数据点紧密分布在回归线两侧
  - 形成了一条清晰的曲线形状
  - 就像：所有数据都沿着一条轨道前进

优点 2：回归线呈现清晰曲线
  - 绿色粗线从 Y=0 平滑下降到 Y=-50
  - 曲线形状符合 30° 弯道的特征
  - 说明：系统能够稳定地跟随曲线

优点 3：置信区间很窄
  - 绿色阴影区域很窄（±5mm 左右）
  - 说明：数据一致性好，每次实验结果相似
  - 就像：每次走同一条弯道，轨迹都很相似

优点 4：曲线平滑
  - 没有突然的转折或跳变
  - 整条曲线很自然流畅
  - 说明：控制系统平滑稳定

为什么 Line Sensor 能做到？
  - 能感知 Leader 的横向位置
  - 提供比例反馈（偏离多少，转多少）
  - 平滑的转向控制 → 平滑的曲线轨迹

【右下图：Bump Sensor 轨迹回归分析】❌ 数据混乱
================================================================================

Bump Sensor 的问题：
--------------------------------------------------
问题 1：点极度分散
  - 数据点散布在 -180mm 到 +180mm 的巨大范围内
  - 看不出明确的曲线形状
  - 就像：数据到处乱飞，完全没有规律

问题 2：回归线不可信
  - 红色粗线勉强画出一条下降趋势
  - 但数据点偏离回归线太远
  - 这条回归线几乎没有参考价值

问题 3：置信区间巨大
  - 红色阴影区域非常宽（±80mm 以上）
  - 说明：数据不一致，每次实验差异巨大
  - 就像：每次走同一条弯道，结果完全不同

问题 4：有反向数据
  - 右上角有很多数据点在正值区域（+150mm）
  - 说明有些实验中机器人转错了方向
  - 应该往下转（负值），却往上转了（正值）

特别严重的问题：
  - 数据分布呈现"对角线分散"模式
  - 不是沿着曲线分布，而是到处散开
  - 说明：Bump Sensor 无法形成稳定的曲线跟随

为什么 Bump Sensor 做不到？
  - 只能感知距离，不能感知方向
  - 无法提前预测曲线
  - 反应式控制 → 总是滞后 → 无法稳定跟随

对比总结：
--------------------------------------------------
数据分散度：Bump 是 Line 的 10 倍以上
回归线质量：Line 清晰，Bump 无意义
置信区间：Bump 宽 15 倍
转错方向：Line 没有，Bump 经常发生

结论：Bump Sensor 在曲线上完全不可用！

【为什么 Line Sensor 在曲线上这么好？】
================================================================================

原理优势：
  1. 横向位置检测：
     - Line：能检测 Leader 向左/右偏移
     - Bump：只知道远近，不知道左右

  2. 预测性控制：
     - Line：提前感知曲线，主动调整
     - Bump：发现偏离后才调整，总是滞后

  3. 比例反馈：
     - Line：偏离越多，转向越大（合理）
     - Bump：转向信号弱，容易判断错误

控制优势：
  1. 平滑转向：Line 的信号变化平滑 → 转向平滑
  2. 方向准确：Line 能判断往哪边转 → 不会转错
  3. 响应及时：Line 能提前响应 → 不会滞后

================================================================================
【为什么 Bump Sensor 在曲线上这么差？】
================================================================================

根本原因：
  1. 缺乏方向信息：
     - 只能测距离，不知道 Leader 往哪边转
     - 就像盲人摸象，只能摸到前面有东西

  2. 反应式控制：
     - 必须等到偏离后才知道要调整
     - 但这时已经来不及了（曲线变化快）

  3. 左右差异小：
     - Bump 的左右传感器都朝前，差异很小
     - 无法提供可靠的转向信号

实验证据：
  - 经常转错方向（正负值混乱）
  - 偏差范围巨大（360mm）
  - 完全无法预测轨迹

简单比喻：
  - 用 Bump Sensor 走曲线 = 蒙着眼开车转弯
  - 只知道前面有东西，不知道弯往哪边
  - 结果就是到处乱撞
