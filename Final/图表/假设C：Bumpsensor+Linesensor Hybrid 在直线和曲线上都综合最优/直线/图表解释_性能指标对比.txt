================================================================================
【上排：Y 位置分布箱线图】
================================================================================

这是什么图？
  - 箱线图（Box Plot）：统计学常用的数据分布图
  - X 轴：实验编号（1-10 次实验）
  - Y 轴：Y 位置（mm）- 机器人在直线上的横向位置
  - 箱子越小 = 数据越集中 = 性能越稳定

如何看懂箱线图？
  - 箱子：中间 50% 的数据集中在这里
  - 箱子中间的横线：中位数
  - 上下两条线（须）：数据的最大/最小值范围
  - 圆点：异常值

左图（红色 - Line Sensor）：极差
--------------------------------------------------
问题 1：箱子巨大
  - 每个实验的箱子高度达 15-30mm
  - 说明每次实验内部数据极度混乱
  - 机器人左右摇摆严重

问题 2：箱子位置混乱
  - 箱子位置完全无规律
  - 有的在 +15mm，有的在 -20mm
  - 无法预测轨迹

问题 3：偏差范围巨大
  - 整体 Y 范围：-50mm 到 +20mm（共 70mm）
  - 在直线任务中，这是灾难性的表现
  - 完全失控

问题 4：大量异常值
  - 很多圆点（异常值）
  - 说明经常出现突发大偏差
  - 系统极不稳定

中图（绿色 - Bump Sensor）：最优秀！
--------------------------------------------------
优点 1：箱子最小！
  - 每个实验的箱子高度仅 2-4mm
  - 是三者中最小的！
  - 说明每次实验内部数据最稳定

优点 2：箱子位置一致！
  - 所有箱子都在 Y=0 附近（-5mm 到 0mm）
  - 位置高度一致，可预测性强
  - 说明能够稳定保持直线

优点 3：偏差范围最小！
  - 整体 Y 范围：-14mm 到 0mm（共 14mm）
  - 是三者中最小的
  - 控制精度最高

优点 4：几乎没有异常值
  - 异常值极少
  - 说明系统极度稳定
  - 很少出现突发偏差

右图（蓝色 - Bump+Line Hybrid）：良好
--------------------------------------------------
优点 1：箱子较小
  - 每个实验的箱子高度约 8-15mm
  - 虽然比 Bump 大，但远小于 Line
  - 说明实验内部数据较稳定

优点 2：箱子位置较一致
  - 大部分箱子在 -10mm 到 0mm 范围内
  - 虽然不如 Bump 集中，但比 Line 好得多
  - 可预测性较好

优点 3：偏差范围适中
  - 整体 Y 范围：-50mm 到 +10mm（共 60mm）
  - 比 Bump 大，但接近 Line
  - 控制性能中等

优点 4：异常值较少
  - 异常值比 Line 少
  - 说明系统较稳定
  - 突发偏差较少

对比总结（上排）：
--------------------------------------------------
箱子大小：Bump（最小）< Hybrid（适中）< Line（最大）
箱子一致性：Bump（最一致）> Hybrid（较一致）> Line（完全混乱）
偏差范围：Bump（14mm）< Hybrid（60mm）≈ Line（70mm）
异常值数量：Bump（最少）< Hybrid（较少）< Line（最多）

结论：Bump 的数据分布最稳定、最一致！
      Hybrid 表现中等，比 Line 好得多
      Line 数据混乱，完全不可靠

【下排：4 个关键性能指标对比】
================================================================================

这是什么？
  - 4 个柱状图：分别展示 4 个关键性能指标
  - 每个图有 3 根柱子：Line（红色）、Bump（绿色）、Hybrid（蓝色）
  - 金色边框：表示该指标的最优表现
  - 数字：具体数值

左上图：Lateral Deviation（横向偏差）- 越小越好
--------------------------------------------------
这是什么指标？
  - 衡量机器人偏离理想直线的程度
  - 越小 = 越贴近直线 = 越好
  - 这是直线跟随的最核心指标！

数据对比：
  Line：13.1 mm（最大）
  Bump：3.6 mm（最小！金色边框）
  Hybrid：12.5 mm（第二大）

分析：
  - Bump 最优：横向偏差最小！
    → Bump 的线性测距能力强
    → 能够精确保持跟随距离
    → 横向偏差仅 3.6mm，几乎完美
  
  - Hybrid 接近 Line：比 Bump 大 3.5 倍
    → 虽然有 Bump 的距离控制
    → 但 Line 部分引入了噪声
    → 导致横向偏差增大
  
  - Line 最差：是 Bump 的 3.6 倍
    → 完全无法控制横向位置
    → 左右摇摆严重

关键发现：
  - Bump 在直线上具有绝对优势！
  - Hybrid 虽然不如 Bump，但仍可接受（12.5mm）
  - Line 完全不适合直线跟随

右上图：Heading Stability（航向稳定性）- 越小越好
--------------------------------------------------
这是什么指标？
  - 衡量机器人航向角的波动范围
  - 越小 = 转向越稳定 = 越好
  - 在直线上，理想情况应该保持 0°

数据对比：
  Line：6.94°（最大）
  Bump：2.07°（最小！金色边框）
  Hybrid：3.95°（第二大）

分析：
  - Bump 最优：航向最稳定！
    → 航向波动仅 2.07°
    → 说明机器人始终保持直线方向
    → 不会左右转向
  
  - Hybrid 第二：比 Bump 大 91%
    → 虽然有 Bump 的稳定性
    → 但 Line 部分会引入转向调整
    → 导致航向波动增大
  
  - Line 最差：是 Bump 的 3.4 倍
    → 航向极不稳定
    → 机器人不停地左右转向
    → 导致轨迹摇摆

关键发现：
  - Bump 能够精确保持直线方向
  - Hybrid 的航向稳定性可接受
  - Line 的航向极不稳定

左下图：Path Straightness（路径直线度）- 越小越好
--------------------------------------------------
这是什么指标？
  - 衡量轨迹偏离理想直线的程度
  - 越小 = 轨迹越直 = 越好
  - 这是直线跟随的关键质量指标

数据对比：
  Line：6.71（最大）
  Bump：1.83（最小！金色边框）
  Hybrid：7.43（最大）

分析：
  - Bump 最优：路径最直！
    → 路径直线度仅 1.83
    → 轨迹几乎是完美直线
    → 这是直线跟随的最佳表现
  
  - Hybrid 最差：比 Bump 大 4.1 倍！
    → 虽然有 Bump 的距离控制
    → 但 Line 部分引入了转向噪声
    → 导致轨迹不如纯 Bump 直
  
  - Line 第二差：比 Bump 大 3.7 倍
    → 路径弯曲严重
    → 完全不是直线

关键发现：
  - Bump 的轨迹最直，这是它的核心优势
  - Hybrid 在这个指标上反而不如预期
    → 这是因为 Line 部分引入了额外的转向调整
    → 在直线上，这些调整反而导致轨迹不够直
  - 但 Hybrid 的综合性能仍然良好（在曲线上表现最优）

为什么 Hybrid 在这个指标最差？
  - Line 部分在直线上信号不稳定
  - 不稳定的信号导致不必要的转向调整
  - 这些调整反而破坏了轨迹的直线性
  - 这是为曲线性能付出的代价

右下图：Repeatability（可重复性）- 越小越好
--------------------------------------------------
这是什么指标？
  - 衡量最终位置的一致性
  - 越小 = 每次实验结果越相似 = 越可靠
  - 这是实际应用中最关键的指标

数据对比：
  Line：24.5 mm（最大）
  Bump：3.8 mm（最小！金色边框）
  Hybrid：18.5 mm（第二大）

分析：
  - Bump 最优：可重复性最强！
    → 每次实验几乎到达同一位置
    → 最终位置偏差仅 3.8mm
    → 系统高度可靠
  
  - Hybrid 第二：比 Bump 大 4.9 倍
    → 虽然有 Bump 的稳定性
    → 但 Line 部分引入的噪声
    → 导致最终位置偏差增大
    → 但仍可接受（18.5mm）
  
  - Line 最差：是 Bump 的 6.4 倍
    → 完全无法预测最终位置
    → 可重复性极差

关键发现：
  - Bump 的可重复性最强，系统最可靠
  - Hybrid 的可重复性可接受
  - Line 完全不可靠

【为什么 Bump 在直线上这么优秀？】
================================================================================

从这张图可以看出：

1. 线性测距能力的优势（核心！）
   - Bump 对距离的测量是线性的
   - 距离变化直接映射到速度控制
   - 横向偏差最小（3.6mm）= 距离控制精度最高

2. 控制简单有效
   - 直线任务只需要保持距离
   - Bump 的控制策略完美适配
   - 航向稳定性最好（2.07°）= 不需要额外转向

3. 信号稳定可靠
   - Bump 的信号不受横向位置影响
   - 抗干扰能力强
   - 路径直线度最好（1.83）= 轨迹最直

4. 可重复性最强
   - 每次实验结果高度一致
   - 系统行为可预测
   - 可重复性最好（3.8mm）= 最可靠

================================================================================
【为什么 Hybrid 在直线上不如 Bump？】
================================================================================

原因分析：

1. Line 部分引入了噪声
   - Line 在直线上信号不稳定（假设 A 已证明）
   - 这些噪声影响转向控制
   - 导致航向波动增大（3.95° vs 2.07°）

2. 时分复用增加了复杂性
   - Bump 和 Line 交替工作
   - 增加了系统响应延迟
   - 不如纯 Bump 简单直接

3. 控制目标的权衡
   - Hybrid 需要兼顾距离和方向
   - 在直线上，方向控制反而引入了干扰
   - 路径直线度反而最差（7.43）

4. 设计理念的权衡
   - Hybrid 的设计目标是"全能"
   - 在直线上牺牲了一些精度
   - 换取了曲线上的最优表现

但 Hybrid 仍然是良好的选择：
  - 虽然不如 Bump，但远超 Line
  - 横向偏差：12.5mm（可接受）
  - 可重复性：18.5mm（可接受）
  - 在曲线上表现最优（置信区间最窄，运动最平滑）
  - 综合全能性最强

================================================================================
【为什么 Line 在直线上这么差？】
================================================================================

根本原因：

1. 测距能力弱
   - Line Sensor 不擅长测量距离
   - 距离估计不准确
   - 无法提供稳定的速度控制
   - 横向偏差大（13.1mm）

2. 信号不稳定
   - 直线上，Leader 始终在正前方
   - 左右传感器信号差异小
   - 噪声比例大
   - 航向极不稳定（6.94°）

3. 控制策略不适配
   - Line 的控制基于横向位置偏差
   - 但直线上横向偏差应该很小
   - 小信号放大后变成大噪声
   - 路径弯曲（6.71）

4. 可重复性差
   - 每次实验结果完全不同
   - 系统行为不可预测
   - 可重复性极差（24.5mm）
